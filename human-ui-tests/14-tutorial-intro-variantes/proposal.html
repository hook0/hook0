Issue #14: Tutorial Intro Layout Variants

3 design directions for the tutorial wizard intro step

Before (screenshot)

Problem: The current intro step stacks: subtitle, illustration, "In this tutorial" text, step list, and skip note -- all vertically with uniform spacing. The illustration sits between the subtitle and the step list, creating a disconnected flow. The "In this tutorial" label floats without clear purpose. The overall layout feels like a list of items rather than an inviting onboarding experience.
Variant 1 -- Compact, no illustration
R
Getting Started

This wizard walks you through creating everything you need to send your first webhook. It takes about 3 minutes.

You will create
  • 1 An organization
  • 2 An application
  • 3 An event type
  • 4 A subscription
  • 5 An event

Prefer to skip? You can also set things up programmatically.

Rationale: Remove the illustration entirely -- it adds visual weight without information. Wrap the step list in a subtle background panel to visually group it. Number the steps instead of using generic icons. Merge subtitle + "In this tutorial" into a single opening paragraph. More compact, more purposeful.
Variant 2 -- Two-column, illustration as accent
R
Getting Started

  • B Create an organization
  • A Create an application
  • T Define an event type
  • L Set up a subscription

Or set up programmatically.

R
Rationale: Keep the illustration but move it to the right as a decorative accent. Content stays left-aligned and scannable. The two-column layout uses horizontal space instead of stacking everything vertically. Illustration is smaller and doesn't interrupt the reading flow.
