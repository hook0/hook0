Issue #25: User Settings Warning Placement

Password change warning renders awkwardly in the card footer

Before (screenshot)

Problem: The Hook0Alert warning is placed inside Hook0CardFooter alongside the submit button, wrapped in a Hook0Stack direction="row" justify="between". This creates an awkward layout: the alert takes up variable width, the button gets pushed to the right, and on narrower screens the row wraps unpredictably. The alert's padded box + the footer's own padding creates excessive visual weight at the bottom of the card.

Proposals

A -- Move warning above the form fields
L
Change Password
! Changing your password will log you out of all other sessions. You will need to log in again.
New password
Confirm password
Move the Hook0Alert from Hook0CardFooter into Hook0CardContent, before the form fields. The warning is the first thing the user reads, setting context before they interact. Footer is clean with just the button, right-aligned. Simple template change: move the alert element up.
B -- Inline subtle warning between fields and button
L
Change Password
New password
Confirm password
! This will log you out of all other sessions.
Replace the full Hook0Alert box with a compact inline warning: just icon + text, no background/border. Place it in the footer above the button in a column layout. Lighter visual weight, reads as a contextual hint rather than a prominent alert. Shorter warning copy.
C -- Warning as card subtitle (recommended)
L
Change Password
This will log you out of all other sessions
New password
Confirm password
Use the existing Hook0CardHeader #subtitle slot for the warning text, styled in the warning color. Remove the Hook0Alert component entirely. The warning becomes part of the section header -- contextually correct (it describes the section's consequence) and visually clean. The subtitle slot already exists and is used in other cards on this page. Indented under the icon to align with the title text.

Implementation Notes

The current code places the Hook0Alert and submit button inside Hook0CardFooter wrapped in Hook0Stack direction="row" justify="between" wrap (lines 230-245 of UserSettings.vue). The wrap prop causes the unpredictable layout on narrow widths. All three proposals remove the alert from the footer, leaving the footer with just the button.
