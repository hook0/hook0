Issue #4: Org Card Header Spacing

Badge misalignment and excessive vertical space in card header

Before (screenshot)

Problem: The org card header has min-height: 70px on .org-card__info, which creates excessive blank space when the org name fits on a single line. The "Developer" badge also appears vertically misaligned from the app count text because .org-card__meta uses align-items: center but the badge has different intrinsic height than the text.

Proposals

A -- Remove min-height, tighten padding
A
Acme Corporation
Developer 1 app
›
Remove min-height: 70px from .org-card__info. Reduce header padding from 0.375rem 0.5rem to 0.25rem 0.375rem. Switch header align: start to align: center so the chevron centers vertically with the content.
B -- Baseline-aligned meta row
A
Acme Corporation
Developer 1 app
›
Same min-height removal. Additionally, change .org-card__meta to align-items: baseline so the badge text and app count text sit on the same typographic baseline. Slightly increase gap to 0.375rem for breathing room.
C -- Inline badge after name
A
Acme Corporation Developer
1 app
›
Move the badge to sit inline with the org name (same row, baseline-aligned). The meta row then only contains the app count, eliminating the alignment problem entirely. Badge wraps to a second line naturally if the org name is long.
