Configuration Mode — Notes de spacing

Problèmes actuels

  • Le label "Configuration mode" et le toggle sont dans un Hook0CardContentLine type split — le padding est trop généreux (1.25rem) pour un simple toggle.
  • La description en dessous est dans un second Hook0CardContentLine type full-width, ce qui ajoute un padding supplémentaire au-dessus.
  • Résultat : espace disproportionné entre le toggle et sa description.

Propositions

A. Fusionner en un seul CardContentLine

Mettre le label + description dans le slot #label (label en haut, description en HelpText en dessous), et le toggle dans #content. Un seul bloc = un seul padding.

B. Réduire le padding du second CardContentLine

Ajouter un type compact ou simplement override le padding-top à 0 sur le second line via une classe scoped.

C. Sortir la description du CardContentLine

Placer le Hook0HelpText de description directement après le toggle CardContentLine, avec un padding horizontal aligné mais un padding vertical réduit (0.5rem au lieu de 1.25rem).

Rendu cible (option A)

Attenuate your token

You can reduce your token's permissions.

Configuration mode

Quickly scope your token to a single application with preset expiry durations.

Recommandation : Option A — un seul CardContentLine avec label+description à gauche et toggle à droite. Supprime l'espace mort.
