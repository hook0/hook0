## MAJOR (consensus findings — 5/5 agents agree)

### M1. CSS class selectors for Hook0Dialog confirm buttons — highest impact

**Consensus: 5/5 agents flagged this.**

The selector `.hook0-dialog .hook0-dialog__actions button:last-child` appears ~15 times across 7+ test files. It is positional (`:last-child`) and coupled to internal CSS classes. A single dialog refactor breaks all CRUD delete/confirm tests simultaneously.

**Files:** `service-tokens.spec.ts` (6x), `service-token-detail.spec.ts`, `subscriptions.spec.ts`, `user-settings.spec.ts`, and others.

**Fix:** Add `data-test="dialog-confirm-button"` and `data-test="dialog-cancel-button"` to `Hook0Dialog.vue`. One component change hardens ~15 tests.

### M6. `page.on("dialog")` handlers are dead code

**Consensus: 4/5 agents flagged this.**

Tests in `service-tokens.spec.ts:246`, `subscriptions.spec.ts:616`, `user-settings.spec.ts:403` register `page.on("dialog", dialog => dialog.dismiss())` for "cancel delete" scenarios. But Hook0 uses a custom `Hook0Dialog` component (DOM modal), not native `window.confirm()`. The handler never fires — the test passes vacuously because the page doesn't navigate regardless.

**Fix:** Remove dead `page.on("dialog")` handlers. For cancel tests, interact with the Hook0Dialog cancel button in the DOM instead.

## MINOR (consensus findings — 3+ agents agree)

### m1. `getByText("Delete")`, `getByText("Edit")`, `getByText("Cancel")` are locale-dependent

**Consensus: 4/5 agents.** Files: `service-token-detail.spec.ts`, `service-tokens.spec.ts`. These selectors match English text and will break if i18n is ever enabled. Add `data-test` to AG Grid cell renderer action buttons.

### m2. Event-sending boilerplate duplicated 5x within events.spec.ts

**Consensus: 4/5 agents.** The send-event sequence (open form, select type, fill labels, set date, submit) is ~20 lines repeated 5 times in the same file. Extract a `sendTestEvent(page, env)` local helper.

### m3. `navigation.spec.ts` sidebar test uses conditional `if (await isVisible())`

**Consensus: 4/5 agents.** Lines 247-280 silently skip assertions if nav links are not visible, masking regressions. Use `await expect(...).toBeVisible()` to fail fast instead.

### m4. `[data-sonner-toast]` is a third-party internal attribute

**Consensus: 3/5 agents.** Used in 6 assertions across 3 files. Coupled to `vue-sonner` internals. Acceptable for now but worth noting.

### m5. `navigation.spec.ts:106-111` `Promise.race` pattern

**Consensus: 3/5 agents.** Could use Playwright's `locator.or()` API instead. Current pattern produces opaque error messages.

### m6. `components-visual.spec.ts` creates full user just to test button CSS

**Consensus: 3/5 agents.** Both tests register + login just to check button flexbox properties on `/settings`. The `button-variants-visual.spec.ts` already covers this via the showcase page without auth. Potentially redundant.

### m7. `[row-id]` AG Grid internal attribute used extensively

**Consensus: 2/5 agents.** Used in all list-page tests. Pragmatic choice since AG Grid is deeply embedded, but fragile if the grid library is ever swapped.

### m8. `networkidle` usage in tutorial wizard flow

**Consensus: 2/5 agents.** `page.reload({ waitUntil: "networkidle" })` at line 136 of `tutorial-wizard-flow.spec.ts`. Playwright docs discourage `networkidle`. Replace with a specific element wait.

## SUGGESTIONS (consensus — 2+ agents)

### S1. Consider Playwright `test.extend` fixtures for authenticated state

**Consensus: 5/5 agents.** Would dramatically reduce boilerplate and test runtime. Single most impactful architectural improvement.

### S2. Consider `test.describe.configure({ mode: 'serial' })` for heavy tests

**Consensus: 3/5 agents.** Tutorial wizard flow and full CRUD tests are expensive. Serial mode within their describe blocks could improve CI reliability.

### S3. Add `test.step()` for better trace readability

**Consensus: 2/5 agents.** Long tests like the wizard flow or navigation mega-test would benefit from named steps in the Playwright trace viewer.

### S4. Visual regression tests may be flaky across browsers/CI

**Consensus: 2/5 agents.** Font rendering differs between macOS and Linux. Consider restricting screenshot tests to a single browser project.

### S5. `registerAndVerifyUser` and `generateTestUser` are dead code

**Consensus: 3/5 agents.** Exported from fixtures but never imported by any test. Either wire them in or remove them.
